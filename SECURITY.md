# Security Policy

## Supported Versions

Currently, as SwarmBox is in the early stages of development, security support is provided on a **best-effort basis**. 
We prioritize addressing critical vulnerabilities that may impact the integrity, safety, and security of the drone swarm framework until further notice.

## Reporting a Vulnerability

We take the security of SwarmBox seriously. If you discover a security vulnerability, please do **not** file a public issue.  
Instead, report it privately using GitHub's built-in vulnerability reporting feature.

### How to Report
1. Please navigate to our [Private Vulnerability Reporting page](https://github.com/postech-compsec/swarmbox/security/advisories/new).
2. Fill out the report form, preferably in **English**. 
3. Include as much detail as possible, such as steps to reproduce the vulnerability and its potential impact on the framework.

### What to Expect
* We will make our best effort to acknowledge your report within a few business days.
* We will work privately with you to investigate, verify, and resolve the issue before any public disclosure is made.
